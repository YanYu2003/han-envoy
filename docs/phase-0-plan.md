# Phase 0：项目定版与工程骨架规划

## 游戏一句话卖点

> 持节出塞，纵横西域。—— 一段 AI 驱动的汉代外交传奇。

## 目标玩家与核心体验

- **目标玩家：** 喜欢历史叙事、策略选择、角色扮演的游戏玩家；对汉代西域历史感兴趣的泛文化人群。
- **核心体验：** 玩家扮演持节汉使，在楼兰朝堂上与 AI 驱动的角色进行外交博弈。每一次选择都影响汉威、胡压、王怒等参数，最终导向不同的结局。

## 参赛定位

- **黑客松赛道：** 赛题三——叙事类游戏
- **竞争优势：** AI 从"噱头"变为"核心循环"——不是用 AI 写文本，而是用 AI 驱动角色反应和自由输入解析
- **技术展示：** 前端 + LLM API 的深度集成

## MVP 范围

| 模块 | 说明 |
|------|------|
| 国家 | 仅做楼兰原型国（不以真实楼兰为名，使用虚构化名但保留历史质感） |
| 场景 | 朝堂场景（入朝 → 辩论 → 决策 → 结局） |
| 角色 | 5–8 个 AI 驱动角色（国王、译者、亲汉大臣、亲胡大臣等） |
| 参数 | 10 项核心外交参数 |
| 输入 | 预设选项 + 自由自然语言输入 |
| 结局 | 至少 3 种不同结局 |

## 核心玩法循环

```
         ┌──────────────────────────┐
         │     玩家进入楼兰朝堂       │
         └──────────┬───────────────┘
                    ▼
         ┌──────────────────────────┐
         │  AI 解析局势 → 国王发话   │
         └──────────┬───────────────┘
                    ▼
         ┌──────────────────────────┐
         │   玩家选择行动方式         │
         │  (预设选项 / 自 由输入)    │
         └──────────┬───────────────┘
                    ▼
         ┌──────────────────────────┐
         │  AI 解析玩家输入 → 参数结算 │
         └──────────┬───────────────┘
                    ▼
         ┌──────────────────────────┐
         │  角色反应（AI 生成）       │
         └──────────┬───────────────┘
                    ▼
         ┌──────────────────────────┐
         │  检查结束条件 → 结局 / 继续 │
         └──────────────────────────┘
```

## 主要系统

### 1. 外交参数系统（硬规则）

10 项核心参数构成游戏状态的骨架：

| 参数 | 中文 | 范围 | 说明 |
|------|------|------|------|
| hanPrestige | 汉威 | 0–100 | 汉朝在楼兰的影响力 |
| xiongnuPressure | 胡压 | 0–100 | 匈奴对楼兰的压力 |
| kingAnger | 王怒 | 0–100 | 国王对使节的愤怒 |
| kingFear | 王惧 | 0–100 | 国王对使节的恐惧 |
| proHan | 亲汉派 | 0–100 | 亲汉派势力值 |
| proXiongnu | 亲胡派 | 0–100 | 亲胡派势力值 |
| tradeAccess | 商道通行 | 0–100 | 商队通行便利度 |
| casusBelli | 战争借口 | 0–100 | 汉朝对楼兰的战争借口充足度 |
| envoyHonor | 使节名节 | 0–100 | 使节个人名节 |
| historianScore | 史官评价 | 0–100 | 史官对使节的最终评价 |

### 2. 玩家行动类型

- **预设选项：** 在每个场景提供若干选项，降低输入门槛
- **自由输入：** 支持自然语言输入，AI 解析为结构化行动
- **行动类型：** 威慑、谈判、忍让、羞辱、离间、要求质子、宣示汉威、刺杀、殉国等

### 3. AI Agent 系统

详见 [ai-agent-design.md](./ai-agent-design.md)

### 4. 结局系统

基于参数阈值触发：

| 结局 | 触发条件 |
|------|----------|
| 功成身退 | 汉威高 + 胡压低 + 使节名节 ≥ 70 |
| 全身而退 | 汉威中 + 王怒低 + 使节名节 ≥ 40 |
| 血溅朝堂 | 王怒 ≥ 80 + 王惧低 |
| 被逐出城 | 王怒 ≥ 60 + 汉威低 |
| 卧薪尝胆 | 使节名节 ≥ 60 + 汉威低 + 王惧高 |
| 身败名裂 | 使节名节 ≤ 20 |
| 自刎殉节 | 玩家选择殉国行动 |
| 刺杀成功/失败 | 玩家选择刺杀行动 + 判定结果 |
| 转投匈奴 | 玩家选择叛国行动（可选做） |

### 5. 第一章楼兰朝堂流程骨架

1. **序幕：** 抵达楼兰城外 + 策马入城 + 在馆驿休整
2. **入宫：** 递交国书 + 观礼 + 初见楼兰王
3. **朝堂辩论：** 轮番发言 + 亲汉/亲胡派交锋 + 疑似匈奴使者现身
4. **关键抉择：** 面对楼兰王的质问/试探，选择回应策略
5. **危机事件：** 随机/参数触发的突发事件（谍报、刺杀、天象等）
6. **结局：** 离宫 + 史官录语 + 使节归宿

## AI Agent 设计边界

- **AI 负责：** 角色对话生成、自由输入意图解析、结局文案美化
- **AI 不负责：** 规则裁决（参数增减）、胜负判定、情节分支路由
- **AI 失败 fallback：** 角色退化为固定句式 + 日志警告
