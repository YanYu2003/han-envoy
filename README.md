# 《汉使》Han Envoy

**持节出塞，纵横西域。一段 AI 驱动的汉代外交传奇。**

---

## 项目背景

《汉使》是一款参加 **「AI CAN DO IT｜腾讯云黑客松 游戏开发挑战赛」** 的网页游戏原型，报名方向为 **赛题三：叙事类游戏**。

作品要求为可独立运行的网页游戏原型，并深度融合 AI 能力。提交内容包括：

- 网页链接
- Demo 视频
- 作品介绍 PPT
- CodeBuddy 历史对话记录

## 参赛定位

| 维度 | 说明 |
|------|------|
| 赛道 | 叙事类游戏 |
| 核心卖点 | AI 驱动的动态外交博弈 + 硬规则参数系统 |
| 文化厚度 | 以汉代使节制度为骨架，不做贴皮 |
| 技术展示 | 前端 + AI Agent 的深度耦合 |

## MVP 范围

- **第一章：楼兰朝堂**（以楼兰/鄯善历史原型为基础）
- 一个完整的朝堂外交场景
- 5–8 个 AI 驱动的角色
- 10 项核心外交参数
- 玩家可通过预设选项或自由输入进行操作
- 至少 3 种不同结局
- 史官评价系统

## 本地运行方式

```bash
# 1. 克隆仓库
git clone https://github.com/<your-org>/han-envoy.git
cd han-envoy

# 2. 安装依赖
npm install

# 3. 配置环境变量（可选，Phase 1 前可跳过）
cp .env.example .env
# 然后编辑 .env 填入 AI API 信息

# 4. 启动开发服务器
npm run dev

# 5. 构建生产版本
npm run build

# 6. 预览构建结果
npm run preview
```

## AI 能力设计说明

AI 在游戏中的角色（详见 [docs/ai-agent-design.md](./docs/ai-agent-design.md)）：

| 能力 | 说明 |
|------|------|
| 自由输入解析 | 将玩家自然语言输入解析为结构化游戏行动 |
| 角色动态反应 | AI 驱动的国王、大臣等角色对玩家行动做出语境化回应 |
| 朝堂辩论 | AI 驱动的多角色对话场景 |
| 结局文案生成 | 根据最终参数状态生成个性化结局描述 |

**设计原则：** AI 负责"内容生成"和"意图理解"，不负责"规则裁决"。所有游戏逻辑的边界由硬规则系统控制。

## API Key 安全说明

- **API Key 仅通过 `.env` 文件配置，不得硬编码在代码中**
- `.env` 文件已在 `.gitignore` 中排除，不会提交到仓库
- 提交前请检查是否意外提交了凭据
- 多人协作时通过加密通道分享 `.env` 内容
