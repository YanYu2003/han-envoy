# Act 0：入朝前准备阶段设计文档

> **当前阶段：Phase 2.5（设计 + 类型预留，尚未接入游戏流程）**

---

## 1. Act 0 的定位

### 为什么需要 Act 0？

当前游戏流程为：

```
开始 → 入朝堂 → 4 场景选择 → 结局
```

这个流程虽然完整但偏短，玩家入朝时对各角色、局势和前任汉使的命运几乎一无所知。Act 0 要解决的问题：

1. **信息不对称：** 玩家入朝前应该有机会了解楼兰局势，而不是盲目选择。
2. **路线铺垫：** 刺王、殉国、问罪等高风险路线需要前置行动才能合理触发。
3. **策略深度：** 有限行动次数的资源管理增加策略感，避免纯线性叙事。
4. **调查线：** 前任汉使的死因既可以作为剧情驱动力，也可以影响参数和结局。

### Act 0 的玩法目标

- 让玩家从"被动接收叙事"变成"主动调查收集"
- 情报和证据影响朝堂中的可用选项和叙事文本
- 让高风险行为（刺王、当庭问罪）不再是一键按钮，而是需要前置准备的策略决策
- 增加调查感和历史沉浸感

### 设计原则

- 有限行动次数（例如 3 次）→ 每次选择都有取舍
- 暴露度系统 → 过于张扬的调查可能打草惊蛇
- 情报 ≠ 证据 → 情报可能不可靠，证据才是决定性力量
- 角色信任 → 多次接触同一角色可能解锁更多信息
- 不强制玩家做完所有准备 → 可以直接入朝，但难度不同

## 2. Act 0 的核心循环

```
┌───────────────────────────────────┐
│  进入 Act 0 准备阶段               │
│  准备行动次数：3/3                 │
│  暴露度：0                         │
└───────────┬───────────────────────┘
            ▼
┌───────────────────────────────────┐
│  选择行动对象或调查方向              │
│  (拜访亲汉贵人 / 调查前任汉使 / ...) │
└───────────┬───────────────────────┘
            ▼
┌───────────────────────────────────┐
│  结算行动效果                      │
│  ├─ 获得情报/证据                  │
│  ├─ 参数变化（如 proHan +5）        │
│  ├─ 暴露度变化                     │
│  ├─ 角色信任变化                   │
│  └─ 解锁后续选项标识                │
└───────────┬───────────────────────┘
            ▼
┌───────────────────────────────────┐
│  剩余行动次数 > 0 且未触发暴露？     │
│      ├─ 是 → 继续选择               │
│      └─ 否 → 进入朝堂              │
└───────────────────────────────────┘
            ▼
┌───────────────────────────────────┐
│  带入朝堂：                        │
│  ├─ 初始参数受准备行动影响           │
│  ├─ 情报/证据解锁新选项              │
│  ├─ 动态叙事反映准备成果             │
│  └─ 结局判定参考调查结果             │
└───────────────────────────────────┘
```

## 3. Act 0 的主要行动类型

### 3.1 拜访亲汉贵人

| 项目 | 说明 |
|------|------|
| 行动目标 | 争取亲汉派支持，了解朝堂内部局势 |
| 可能获得情报 | 亲匈派势力格局、楼兰王近期态度、王庭礼仪信息 |
| 可能获得证据 | 亲汉贵人提供的书面证词或密信 |
| 参数影响 | proHan +5–10, kingAnger +3（若被察觉）, exposure +5–10 |
| 对朝堂影响 | 解锁"支持亲汉派"选项的增强版本，增加初始 proHan |

### 3.2 拉拢译者

| 项目 | 说明 |
|------|------|
| 行动目标 | 争取译者至少在翻译中不做手脚 |
| 可能获得情报 | 朝堂关键人物性格、前任汉使被翻译扭曲的证词 |
| 可能获得证据 | 译者提供的线索或前任汉使遗言 |
| 参数影响 | 译者信任 +15–25, exposure +5 |
| 对朝堂影响 | 未来可实现译者不曲译、提示关键信息 |

### 3.3 贿赂王庭近侍

| 项目 | 说明 |
|------|------|
| 行动目标 | 通过内线获取王庭内部情报 |
| 可能获得情报 | 楼兰王近期决策、亲匈派密谋内容、王庭守卫轮换 |
| 可能获得证据 | 密信副本、亲匈派受贿记录 |
| 参数影响 | hanPrestige -5（若败露）, proHan +5, exposure +10–15 |
| 对朝堂影响 | 解锁"威胁亲匈大臣"选项的增强版，提供额外叙事信息 |

### 3.4 调查前任汉使死因

详见第 4 节。

## 4. 前任汉使死因调查线

### 4.1 为什么设计这条线

前任汉使的死是楼兰与汉朝关系中的关键事件。调查它能够：

- 为玩家的行动提供情感驱动力
- 提供「当庭问罪」的正当性
- 影响 casusBelli（战争借口）
- 影响 envoyHonor（使节名节）
- 决定某些结局的走向（如殉国是否转化为战略胜利）

### 4.2 调查支线设计

调查分为多个方向和深度，不同方向获得不同信息：

#### 方向 A：亲匈派陷害

| 信息 | 说明 |
|------|------|
| 线索 | 译者暗示前任汉使之死另有隐情 |
| 关键证据 | 亲匈大臣受贿账册（显示他收了匈奴单于的黄金来策划"汉使意外"） |
| 参数影响 | casusBelli +15, proXiongnu +10（暴露则增加）, proHan +5 |
| 解锁选项 | 朝堂上"当庭问罪"选项解锁/增强 |

#### 方向 B：楼兰王默许

| 信息 | 说明 |
|------|------|
| 线索 | 近侍透露前任汉使死前曾被密召入偏殿 |
| 关键证据 | 近侍证词（楼兰王默许亲匈派动手） |
| 参数影响 | kingAnger +10, casusBelli +10, envoyHonor +5 |
| 解锁选项 | 朝堂上可直接质问楼兰王，触发特殊叙事分支 |

#### 方向 C：译者曾被迫隐瞒

| 信息 | 说明 |
|------|------|
| 线索 | 译者拒绝谈论前任汉使，行为反常 |
| 关键证据 | 译者密证（亲匈大臣威胁他篡改翻译，导致汉使误解被杀） |
| 参数影响 | 译者信任 +20, casusBelli +8, proXiongnu +5 |
| 解锁选项 | 朝堂上译者暗中提示关键信息 |

#### 方向 D：找到汉使遗物

| 信息 | 说明 |
|------|------|
| 线索 | 亲汉贵人暗中保留了前任汉使的遗物 |
| 关键证据 | 残破旌节 + 血书残片（汉使死前写下的名字） |
| 参数影响 | envoyHonor +10, casusBelli +10, historianScore +5 |
| 解锁选项 | 朝堂上展示物证，增强"责背约"选项的效果 |

#### 查无实据

如果玩家选择了错误的方向或行动次数不足，可能只得到模糊传闻，无法获得实质性证据。此时：

- casusBelli 不额外增长
- 朝堂上无法触发"当庭问罪"分支
- 但玩家仍然可以根据推断自由行动

### 4.3 调查结果对系统的具体影响

| 系统模块 | 影响方式 |
|----------|----------|
| casusBelli | 找到证据直接增加，查无实据则不增加 |
| envoyHonor | 找到使节遗物增加，成功为前任复仇更增名节 |
| kingAnger | 若暴露调查意图则增加，证据越硬增加越少 |
| kingFear | 调查出真相后间接增加（汉使死因暴露 = 汉朝可能报复） |
| proHan | 亲匈派陷害证据 → 亲汉势力增长 |
| proXiongnu | 被察觉调查则增长，提前警觉 |
| 当庭问罪选项 | 需要关键证据才能解锁 |
| 殉国结局 | 如果有前任汉使的教训，玩家殉国时可以选择"吸取教训"分支 |
| 刺王结局 | 有调查铺垫的刺王路线更完整，评判时更合理 |
| 史官评价 | 尽职调查 → 史官记录"使者明察秋毫" |

## 5. 新增资源系统设计

### 5.1 资源一览

| 资源 | 说明 | 范围 | 初始值 |
|------|------|------|--------|
| intel（情报） | 收集到的信息片段，可能可靠或不可靠 | 集合 | 空 |
| evidence（证据） | 确凿的证据物品，可用于朝堂 | 集合 | 空 |
| contactTrust（角色信任） | 每个可接触角色对玩家的信任度 | 0–100 | 初始值 |
| exposure（暴露度） | 玩家调查行动被察觉的程度 | 0–100 | 0 |
| preparationActionsRemaining（剩余行动次数） | 玩家还能进行的准备行动次数 | 0–3 | 3 |

### 5.2 各资源的作用

#### 情报（Intel）

- 提供场景内叙事信息，增强玩家决策依据
- 不一定可靠——来源不同的情报可能互相矛盾
- 纯情报不会直接改变参数，但可能指引玩家找到证据

类型示例：
```
{ id: "intel_xiongnu_letter", title: "亲匈大臣通敌密信", 
  description: "左将近期与匈奴使者秘密会面三次", 
  reliability: 0.7, source: "近侍", tags: ["xiongnu", "conspiracy"] }
```

#### 证据（Evidence）

- 可在朝堂上公开使用的确凿物品
- 解锁特定选项（如"当庭问罪"）
- 被出示后永久消耗或保留（取决于证据类型）
- 影响结局判定和史官评价

类型示例：
```
{ id: "evidence_broken_jie", title: "残破旌节", 
  description: "前任汉使被折断的旌节，上面有暗红色血迹", 
  strength: 0.9, canUseInCourt: true, 
  unlockChoiceIds: ["accuse_in_court_strong"] }
```

#### 角色信任（ContactTrust）

- 影响角色愿意透露的信息深度
- 高信任 = 更关键的情报/证据
- 某些行动（贿赂、威胁）可能降低信任但快速获得信息
- 译者信任阈值：≥50 透露基本线索，≥75 透露关键证词

#### 暴露度（Exposure）

- 暴露度高 → 楼兰王和亲匈派对玩家产生戒心
- 暴露度 ≥ 60 → 王庭开始监视玩家
- 暴露度 ≥ 80 → 部分行动直接不可用，朝堂初始参数恶化
- 暴露度在每个 Act 0 行动后自然增长，高风险行动增长更多

暴露度对朝堂的影响：
```
exposure ≥ 40: kingAnger +5, 开始部分行动无法进行
exposure ≥ 60: kingAnger +10, proXiongnu +5
exposure ≥ 80: kingAnger +15, 所有"秘密接触"类选项锁定
```

#### 剩余准备行动次数

- 固定 3 次，不能增加
- 鼓励玩家做出取舍——不可能所有方向都覆盖
- 允许提前入朝（主动结束准备阶段），但会损失未用完的次数

## 6. 与现有系统的关系

### 6.1 如何改变初始参数（initialStats）

Act 0 中每个行动的效果叠加后，修正 `initialState.ts` 中的默认值：

```typescript
// 示例：如果玩家拜访了亲汉贵人并获得了支持
// initialStats 变为：
{
  ...INITIAL_STATS,
  proHan: INITIAL_STATS.proHan + 10,  // +10 from visit
  hanPrestige: INITIAL_STATS.hanPrestige + 3,
}
```

### 6.2 如何解锁或禁用 choice

在 `Choice.condition` 中增加对证据的检查：

```typescript
// choice id: "accuse_in_court"
condition: (stats: GameStats, preCourtState?: PreCourtState) => {
  return preCourtState?.evidenceIds.includes("evidence_blood_letter") ?? false;
}
```

### 6.3 如何添加动态叙事

朝堂场景的 `narrativeVariants` 增加对证据和情报的检查：

```typescript
// 在 intro_court 场景中
(s) => preCourtState?.intelIds.includes("intel_king_fear") 
  ? "你注意到楼兰王的眼中闪过一丝不易察觉的紧张——你早知道他会这样。"
  : ""
```

### 6.4 如何影响阈值事件

- 如果玩家获得了关键证据，某些阈值事件可能提前触发（例如 casusBelli ≥ 70 更容易达成）
- 如果暴露度过高，可能提前触发"王怒滔天"事件

### 6.5 如何影响结局判定

- 调查充分的玩家在刺王、殉国路线中有更有利的结局判定
- 例如刺王成功结局：若无调查铺垫 → "兵行险招"；有调查铺垫 → "谋定后动"
- 史官评价在 Act 0 中有所作为的玩家额外加分

### 6.6 如何让刺王成为可准备路线

当前"冒险刺王"是在最后一刻突然出现的按钮。Act 0 将其改造为：

1. **准备阶段：** 探查守卫 + 准备短刃 + 秘密接触内应
2. **朝堂阶段：** 选择"发动刺王"时，效果根据准备程度动态变化
3. **准备充分时：** 更大的成功率，更低的负面后果
4. **准备不足时：** 当前 Phase 2 的默认效果（高风险，成败皆有可能）

## 7. 后续实现路线

| Phase | 内容 |
|-------|------|
| **Phase 3** | Mock AI + 自由输入 + 角色反应。暂不实现 Act 0 |
| **Phase 4** | 真实 AI 接入。暂不实现 Act 0 |
| **Phase 5** | 正式实现 Act 0 入朝前准备阶段（UI + Store + 数据接入） |
| **Phase 6** | 美术、部署、Demo、PPT |
| **Post-MVP** | 为 Act 0 增加更多行动类型、更复杂调查线、多结局关联 |

### 为什么不在 Phase 3/4 实现 Act 0

1. 先完成核心朝堂外交体验（Mock AI → 真实 AI）
2. Act 0 需要完整的 UI（PreCourtScreen）和全新的 store 逻辑
3. 在朝堂玩法稳定后再扩展前奏，更有利于测试和迭代
4. 黑客松进度安排：优先保证核心玩法（朝堂 + AI）的演示效果
