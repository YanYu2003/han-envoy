# 《汉使》游戏设计文档

## 1. 核心玩法循环

### 宏观循环

```
           ┌───────────────────────┐
           │     章节开始           │  ← 每个章节 = 一个国家
           └─────────┬─────────────┘
                     ▼
           ┌───────────────────────┐
           │   到达 → 入宫 → 朝堂    │  ← 场景序列
           └─────────┬─────────────┘
                     ▼
           ┌───────────────────────┐
           │   回合制外交博弈        │  ← 每个回合 = 玩家行动 + AI 反应 + 参数结算
           └─────────┬─────────────┘
                     ▼
           ┌───────────────────────┐
           │   检查结局条件          │
           │   参数触发 / 事件触发   │
           └─────────┬─────────────┘
                     ▼
           ┌───────────────────────┐
           │   结局展示 + 史官评语   │
           └───────────────────────┘
```

### 微观回合流程

每个回合分 4 步：

1. **局势简报：** 当前参数状态 + 场景描述 + 角色神态
2. **玩家决策：** 显示 3–4 个预设选项，或选择自由输入
3. **AI 处理：**
   - 若为预设选项 → 直接映射为行动 + 参数变动
   - 若为自由输入 → AI 解析为结构化行动，再映射到参数变动
4. **结算反馈：**
   - 参数变动显示
   - 角色反应对话（AI 生成）
   - 检查是否触发结局/特殊事件

## 2. 外交参数系统

### 核心参数（硬规则）

所有参数为 0–100 整数，初始值根据不同场景和难度设定。

```typescript
interface GameState {
  hanPrestige: number;       // 汉威 — 汉朝在楼兰的影响力
  xiongnuPressure: number;   // 胡压 — 匈奴对楼兰的压力
  kingAnger: number;         // 王怒 — 国王对使节的愤怒
  kingFear: number;          // 王惧 — 国王对使节的恐惧
  proHan: number;            // 亲汉派 — 亲汉势力值
  proXiongnu: number;        // 亲胡派 — 亲胡势力值
  tradeAccess: number;       // 商道通行 — 商队通行便利度
  casusBelli: number;        // 战争借口 — 汉朝对楼兰的战争借口
  envoyHonor: number;        // 使节名节 — 使节个人名节
  historianScore: number;    // 史官评价 — 最终评价
}
```

### 参数影响关系

```
玩家行动
    │
    ▼
┌─────────────────────────────────────────────┐
│  直接参数变动（硬规则表）                      │
│                                              │
│  威慑 → hanPrestige +5, kingFear +10,       │
│         kingAnger +10, envoyHonor +3         │
│                                              │
│  谈判 → hanPrestige +2, kingAnger -5,       │
│         tradeAccess +5, envoyHonor +5        │
│                                              │
│  忍让 → kingAnger -10, hanPrestige -5,      │
│         tradeAccess +3, envoyHonor -3         │
│                                              │
│  羞辱 → kingAnger +20, kingFear +5,         │
│         proXiongnu +10, envoyHonor -5        │
│                                              │
│  离间 → proHan +10, proXiongnu -10,         │
│         hanPrestige +5, envoyHonor +3        │
│                                              │
│  宣示汉威 → hanPrestige +10, kingFear +15,  │
│             kingAnger +5, proHan +5          │
│                                              │
│  要求质子 → hanPrestige +8, kingAnger +15,  │
│             kingFear +20, proXiongnu +10     │
└─────────────────────────────────────────────┘
                               │
                               ▼
                     ┌─────────────────┐
                     │ 参数连锁反应     │
                     │（自动结算每回合）│
                     │                 │
                     │ 王怒 > 王惧      │
                     │   → 进攻倾向    │
                     │ 王惧 > 王怒      │
                     │   → 屈服倾向    │
                     │ hanPrestige 高   │
                     │   → 亲汉派增长   │
                     │ xiongnuPressure 高│
                     │   → 亲胡派增长   │
                     └─────────────────┘
```

### 参数阈值事件

| 参数 | 阈值 | 触发事件 |
|------|------|----------|
| kingAnger ≥ 80 | 王怒爆发 | 进入"血溅朝堂"事件线 |
| kingFear ≥ 80 | 王惧溢出 | 国王可能屈服/逃跑 |
| kingAnger ≥ 60 + kingFear ≤ 30 | 王怒 > 王惧 | 国王可能下令囚禁或处死使节 |
| proHan ≥ 70 | 亲汉倒向 | 亲汉派可能发动政变 |
| proXiongnu ≥ 70 | 亲胡倒向 | 亲胡派可能建议国王投靠匈奴 |
| casusBelli ≥ 60 | 战争借口充足 | 触发"大军压境"事件 |
| envoyHonor ≤ 20 | 名节扫地 | 触发"身败名裂"警告 |
| envoyHonor ≥ 80 | 名节高洁 | 触发"名垂青史"提示 |

## 3. 玩家行动类型

### 预设选项

每个场景提供 3–4 个预设选项，例如：

在楼兰王的试探中：
1. 「大汉天子威加四海，楼兰若能归附，必有重赏。」（宣示汉威 — 强硬的威慑）
2. 「楼兰居中平衡，汉亦不勉强，唯愿商道畅通。」（谈判 — 务实的协商）
3. 「匈奴能给的，大汉也能给，且给的更多。」（谈判 — 利益诱惑）
4. 自由输入（玩家自行输入言论或行动）

### 自由输入行动类型（AI 解析结果）

所有自由输入都会被 AI 解析为以下之一：

| intent | 中文 | 说明 |
|--------|------|------|
| threaten | 威慑 | 以武力或制裁威胁 |
| negotiate | 谈判 | 商议利益交换 |
| appease | 忍让 | 退让以缓和关系 |
| insult | 羞辱 | 当面侮辱 |
| divide | 离间 | 挑拨君臣关系 |
| demand_hostage | 要求质子 | 要求送王子为质 |
| invoke_han_authority | 宣示汉威 | 强调汉朝权威 |
| assassinate | 刺杀 | 刺杀目标角色 |
| surrender | 屈服 | 放弃立场 |
| martyrdom | 殉国 | 以身殉节 |
| unclear | 不明 | AI 无法解析 |

## 4. 角色系统

### 楼兰朝堂角色

| 角色 | 身份 | AI 驱动 | 关键属性 |
|------|------|---------|----------|
| 楼兰王 | 国王 · 中年 | ✅ | 多疑、精明、在汉匈之间摇摆 |
| 左大将 | 亲匈派首领 · 武将 | ✅ | 强硬、亲匈奴、对汉使敌视 |
| 贵人 | 亲汉派首领 · 文臣 | ✅ | 温和、亲汉、暗中倾向于汉朝 |
| 通译 | 翻译官 · 中间人 | ✅ | 油滑、见风使舵、掌控信息 |
| 匈奴使者 | （剧情伏笔） | ❌ | 场景化出场，非核心角色 |
| 汉副使 | 使团副手 | ❌ | 提供建议和情报 |
| 卫士/仆从 | 场景背景 | ❌ | 氛围用 |

### 角色 AI 行为逻辑

每个 AI 角色拥有：
- **立场倾向：** proHan 或 proXiongnu 的镜像
- **个性标签：** 如多疑、温和、暴躁、狡诈
- **状态影响：** 当前参数改变其行为和对话风格
- **记忆：** 对玩家历史行动有一定记忆（长文本上下文）

## 5. 结局系统

### 结局触发逻辑

```
每回合结算后：
  → 检查各参数阈值
  → 若满足结局条件 → 触发出结局
  → 否则继续下一个回合
  → 若回合数 > MAX_ROUND (设为 10) → 触发"时间耗尽"结局
```

### 结局列表（MVP 版本至少实现 3 个）

| 结局 | 条件 | 史官评价 |
|------|------|----------|
| ✅ 功成身退 | hanPrestige ≥ 70, envoyHonor ≥ 70 | 「使不辱命，国威远扬」 |
| ✅ 全身而退 | hanPrestige ≥ 40, envoyHonor ≥ 40, kingAnger ≤ 50 | 「全身而归，无功无过」 |
| ✅ 血溅朝堂 | kingAnger ≥ 80, ENVOY_HONOR ≥ 50 | 「使节殉节，血染塞外」 |
| 被逐出城 | kingAnger ≥ 60, hanPrestige ≤ 30 | 「狼狈出城，汉威扫地」 |
| 身败名裂 | envoyHonor ≤ 20 | 「屈膝折节，史笔如铁」 |
| 刺杀成功 | 玩家选择刺杀且判定成功 | 「一刺定乾坤，功过后人评」 |
| 自刎殉节 | 玩家选择殉国 | 「伏剑而死，气节长存」 |

## 6. 第一章楼兰朝堂流程

### Act 1：入城（序幕）

- 玩家率使团抵达楼兰城外
- 场景描写：西域风貌、残阳、烽燧
- 通译出城迎接，礼节性对话
- **小提示：** 城门礼节的选择影响初始参数

### Act 2：入宫（递交国书）

- 进入楼兰宫内
- 递交国书仪式（礼节选项影响 hanPrestige + 王怒）
- 楼兰王初见，言语试探
- 亲汉派（贵人）暗中使眼色，亲匈派（左大将）面露不屑
- **第一个关键选项：** 面对楼兰王的开场问候，如何回应？

### Act 3：朝堂辩论（核心回合循环）

- 楼兰王提出议题（通常涉及汉匈关系）
- 按回合制进行辩论：
  - 每回合：玩家发言或行动 → AI 角色反应 → 参数结算
  - 亲汉派和亲匈派穿插发言
  - 通译可能扭曲或缓和部分言辞
- **隐藏事件：** 匈奴使者在辩论中途现身，施加压力

### Act 4：危机与抉择

- 某一回合中出现危机事件（取决于参数走向）：
  - 王怒高 → 卫兵围上，气氛紧张
  - 亲胡派占优 → 匈奴使者被请上座
  - 王惧高 → 楼兰王私下召见，请求汉朝庇护
- 玩家做出关键抉择，决定结局走向

### Act 5：结局

- 离宫 / 逃亡 / 殉国
- 史官录语
- 使节归宿（归途、留驻、战死等）
- 展示最终参数面板
- （可选）彩蛋：下一章预告
